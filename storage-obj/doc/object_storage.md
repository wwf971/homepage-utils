# Object Storage (Versioned, Space-Isolated)

## Overview

This module is a Dockerized middle layer between app backends and PostgreSQL.

It stores versioned objects with payload type:
- `text`
- `bytes`
- `json`

---

## Core Concepts

- **space isolation**: each space has independent object tables and metadata table.
- **status model**: each object type has a status table with `versionIdHead` and `isDeleted`.
- **history tree model**: object versions form a version tree; `versionIdHead` (headVersionId concept) points to the current checked-out node.
- **checked-out current copy**: the current HEAD version data is materialized in current object table.
- **history durability with optional data release**: history rows can keep trace metadata while deleting payload (`isDataDeleted = true`).
- **immutable version rows**: a version row is not editable after creation.
- **append-only update**: update means appending a new version from current HEAD, then moving `versionIdHead` to the new version.

---

## ID Strategy

### `objectId` and `spaceId`

- generated by `ms_48` strategy (same as version IDs)
- globally unique
- non-semantic identity IDs
- stored as `bigint` in database

`spaceId` is identity only. Human-readable space name should be stored in metadata.

### `versionId`

- generated by `ms_48` strategy (see `@project/2025/homepage-utils/doc/id.md`)
- globally unique across all objects
- non-sequential per object (tree-friendly history model)
- stored as `bigint`
- same strategy for `versionIdPrev` and `versionIdHead`

### `ms_48` schema (copied)

- 64-bit timestamp-based ID: `48-bit unix ms` + `16-bit offset`
- high 48 bits: milliseconds since epoch (supports timestamp range until year 8921)
- low 16 bits: per-millisecond offset/counter (up to 65,536 IDs per millisecond)

```
# 64-bit id based on unix timestamp
high |-------48bit------|---16bit---| low
high |---unix_stamp_ms--|---offset--| low
```

## Table Structure

### 1) Global metadata table

`metadata`

| Column | Type | Notes |
|--------|------|-------|
| `tag` | `text` PK | metadata key |
| `valueType` | `smallint` | uint8 semantic |
| `valueText` | `text` | nullable |
| `valueJson` | `jsonb` | nullable |
| `valueBytes` | `bytea` | nullable |
| `valueInt` | `bigint` | nullable |
| `valueBoolean` | `boolean` | nullable |
| `updatedAt` | `timestamptz` | default `now()` |
| `updatedAtTz` | `varchar(6)` | nullable, `±HH:MM` |

Conventional tags (all optional; when present, format must match):
- `spacesIdList`
  - `valueType = 2` (json)
  - `valueJson = [<spaceId bigint>, ...]`
- `spaceNameById:{spaceId}`
  - `valueType = 1` (text)
  - `valueText = "<human-readable name>"`
- `spaceCreateAtById:{spaceId}`
  - `valueType = 1` (text) or `4` (int)
  - value is creation time for the space
- `spaceCreateAtTzById:{spaceId}`
  - `valueType = 1` (text)
  - `valueText` format `±HH:MM`

### 2) Global change log table

`change_log`

| Column | Type | Notes |
|--------|------|-------|
| `changeLogId` | `bigserial` PK | |
| `createdAt` | `timestamptz` | default `now()` |
| `createdAtTz` | `varchar(6)` | nullable, `±HH:MM` |
| `commentText` | `text` | nullable |

### 3) Per-space tables

For each `spaceId`, create these tables.

Because `spaceId` is numeric (`bigint`), physical table names should use prefix `space_{spaceId}_`.

Tables:

- `space_{spaceId}_object_text`
- `space_{spaceId}_object_bytes`
- `space_{spaceId}_object_json`
- `space_{spaceId}_object_text_history`
- `space_{spaceId}_object_bytes_history`
- `space_{spaceId}_object_json_history`
- `space_{spaceId}_object_text_status`
- `space_{spaceId}_object_bytes_status`
- `space_{spaceId}_object_json_status`
- `space_{spaceId}_metadata`

Example (spaceId = `281474976710657`):
- `space_281474976710657_object_text`
- `space_281474976710657_object_text_history`
- `space_281474976710657_object_text_status`
- `space_281474976710657_metadata`

#### Current tables (`space_{spaceId}_object_text`, `space_{spaceId}_object_bytes`, `space_{spaceId}_object_json`)

Current tables contain checked-out HEAD payload only for non-deleted objects.

| Column | Type | Notes |
|--------|------|-------|
| `objectId` | `bigint` PK | ms_48 ID |
| `versionId` | `bigint` | ms_48 version ID, same as HEAD |
| `valueText` / `valueBytes` / `valueJson` | `text` / `bytea` / `jsonb` | per table type |
| `changeLogId` | `bigint` FK -> `change_log.changeLogId` | |
| `createdAt` | `timestamptz` | default `now()` |
| `createdAtTz` | `varchar(6)` | nullable |
| `updatedAt` | `timestamptz` | default `now()` |
| `updatedAtTz` | `varchar(6)` | nullable |

Deleted objects should not appear in current tables.

#### History tables (`space_{spaceId}_object_*_history`)

History tables are append-only version trace rows.

| Column | Type | Notes |
|--------|------|-------|
| `objectId` | `bigint` | |
| `versionId` | `bigint` | ms_48 version ID |
| `versionIdPrev` | `bigint` | nullable, ms_48 version ID |
| `valueText` / `valueBytes` / `valueJson` | `text` / `bytea` / `jsonb` | nullable |
| `isDataDeleted` | `boolean` | default `false` |
| `changeLogId` | `bigint` FK -> `change_log.changeLogId` | |
| `createdAt` | `timestamptz` | default `now()` |
| `createdAtTz` | `varchar(6)` | nullable |

Primary key:
- (`objectId`, `versionId`)

When releasing history payload data:
- set payload column to `null`
- set `isDataDeleted = true`
- keep version trace fields unchanged

#### Status tables (`space_{spaceId}_object_*_status`)

Auxiliary object status per object.

| Column | Type | Notes |
|--------|------|-------|
| `objectId` | `bigint` PK | |
| `versionIdHead` | `bigint` | current HEAD version (ms_48) |
| `isDeleted` | `boolean` | delete state for object |
| `changeLogId` | `bigint` FK -> `change_log.changeLogId` | |
| `updatedAt` | `timestamptz` | default `now()` |
| `updatedAtTz` | `varchar(6)` | nullable |

#### Per-space metadata table (`space_{spaceId}_metadata`)

Columns are same as global `metadata` table.

| valueType | Meaning | Active value column |
|-----------|---------|---------------------|

| `3` | bytes | `valueBytes` |

Only one value column should be populated according to `valueType`.

## Version Flow

### Create object

1. Allocate initial version (for example `1`).
2. Insert version row into `space_{spaceId}_object_*_history`.
3. Insert same version into `space_{spaceId}_object_*` (checked-out current copy).
4. Insert/update `space_{spaceId}_object_*_status`:
   - `versionIdHead = <new version>`
   - `isDeleted = false`

### Update object

1. Read HEAD from `space_{spaceId}_object_*_status`.
2. Allocate next version.
3. Insert new version row into history table.
4. Upsert checked-out row in current table to that version.
5. Update status table (`versionIdHead`, `isDeleted = false`).

Note:
- Existing version rows are immutable.
- Update never mutates old version rows; it always appends a new version.

### Delete object (soft delete)

1. Keep status row and set `isDeleted = true`.
2. Keep `versionIdHead` unchanged (or move to a new delete-marker version based on policy).
3. Remove row from current table.

### Rollback to history version

1. Read target history version.
2. Update HEAD to target version (`versionIdHead = targetVersionId`).
3. Set `isDeleted = false`.
4. Check out target payload into current table.

This operation is a checkout of an existing history node to current.

## API Endpoints

IDs should be passed through query params (`GET`) or request body (`POST`), not URL path variables.

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/objectStorage/health/test` | Test PostgreSQL connection |
| GET | `/objectStorage/space/list` | List all spaces from `metadata` (`spacesIdList`) |
| POST | `/objectStorage/space/create` | Create new `spaceId` (ms_48) and initialize `space_{spaceId}_*` tables |
| POST | `/objectStorage/space/delete` | Delete or disable a space |
| POST | `/objectStorage/space/metadata/set` | Set one metadata entry in `space_{spaceId}_metadata` |
| GET | `/objectStorage/space/metadata/get` | Get one metadata entry from `space_{spaceId}_metadata` |
| GET | `/objectStorage/space/metadata/list` | List metadata entries from `space_{spaceId}_metadata` |
| POST | `/objectStorage/metadata/set` | Set one global metadata entry in `metadata` |
| GET | `/objectStorage/metadata/get` | Get one global metadata entry from `metadata` |
| GET | `/objectStorage/metadata/list` | List global metadata entries from `metadata` |
| POST | `/objectStorage/object/create` | Create object and first version (history + current + head) |
| GET | `/objectStorage/object/get` | Get checked-out current object payload |
| GET | `/objectStorage/object/status/get` | Get object status (`versionIdHead`, `isDeleted`) |
| POST | `/objectStorage/object/update` | Create new version and move HEAD |
| POST | `/objectStorage/object/delete` | Mark `isDeleted=true` and remove current row |
| POST | `/objectStorage/object/restore` | Set `isDeleted=false` and checkout `versionIdHead` |
| GET | `/objectStorage/object/version/list` | List history versions of object |
| GET | `/objectStorage/object/version/get` | Get one history version payload/trace |
| POST | `/objectStorage/object/version/checkout` | Check out target history version as current (`versionIdHead = versionId`) |
| POST | `/objectStorage/object/version/rollback` | Alias of checkout for compatibility |
| POST | `/objectStorage/object/version/data/delete` | Delete payload from one history row (`isDataDeleted=true`) |
| GET | `/objectStorage/object/list` | List objects by space/type/deleted status (paged) |

## Request Notes

- Object APIs require:
  - `spaceId`
  - `payloadType` (`text` | `bytes` | `json`)
  - `objectId` (except create when server can allocate)
- `spaceId`, `objectId`, `versionId*` are all ms_48 IDs (`bigint` in DB)
- Payload fields:
  - text: `valueText`
  - bytes: `valueBase64` (API decodes to `bytea`)
  - json: `valueJson`
- Change comment fields:
  - `commentText` (nullable)
  - `commentJson` (nullable)
- Timezone input fields (optional where relevant):
  - `createdAtTz`
  - `updatedAtTz`
  - format `±HH:MM`

## SQL Constraint Suggestions

- check `objectId > 0`
- check `spaceId > 0`
- check `versionId > 0` for current/history/status version fields
- deleted objects must not have row in current table (enforced by service logic)
- FK `changeLogId -> change_log.changeLogId`
- metadata one-value-column-only rule by `valueType`

## Design Principles (Small)

### Database design

- table names use `snake_case`
- column names use `camelCase`
- for every `...At` column, add paired nullable timezone column `...AtTz`
- timezone format in `...AtTz`: `±HH:MM` (minute precision), for example `+09:00`, `-05:30`

### API design

- IDs are passed in query parameters (`GET`) or request body (`POST`), not URL path variables
- version rows are immutable
- update is append-only and advances `versionIdHead`
- `spaceId`, `objectId`, and version IDs use `ms_48` (global unique, tree-friendly), stored as `bigint`
